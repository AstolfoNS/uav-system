## 1. 文档目的

本文档用于指导 `uav-system` 在开发环境与生产环境中的安装、配置、启动、联调与常见故障处理。

适用范围：

- Windows 本地开发
- Linux 服务器部署
- 三服务联调（frontend + backend-core + yolo-service）

---

## 2. 项目组成与启动顺序

项目由三个核心模块组成（统一位于 `projects/` 目录下）：

1. `projects/yolo-service`（FastAPI，默认 `8000`）
2. `projects/backend-core`（Spring Boot，默认 `8111`，context-path `/api/v1`）
3. `projects/uav-frontend`（Vite，默认 `5173`）

推荐启动顺序：

1. 基础设施：MySQL / Redis / MinIO
2. `yolo-service`
3. `backend-core`
4. `uav-frontend`

---

## 3. 环境要求

## 3.1 必备软件

- Git
- Node.js（建议 LTS 版本）
- npm（随 Node.js）
- Java JDK 25（与 `projects/backend-core/pom.xml` 保持一致）
- Maven（可选，项目自带 `mvnw`）
- Python 3.10+
- uv（用于 yolo-service 依赖管理）
- MySQL 8+
- Redis 6+
- MinIO

## 3.2 端口建议

- 前端：`5173`
- 后端：`8111`
- 算法服务：`8000`
- MySQL：`3306`
- Redis：`6379`
- MinIO：`9000`

## 3.3 推荐版本（避免兼容性问题）

- Node.js：建议 `20.x LTS`
- Java：建议与 `projects/backend-core/pom.xml` 一致
- Python：建议 `3.10` 或 `3.11`
- MySQL：建议 `8.0+`
- Redis：建议 `6.2+`

---

## 4. 拉取代码

在目标机器执行：

```bash
git clone https://git.example.com/your-team/uav-system.git uav-system
cd uav-system
```

说明：

- 上述地址为示例，请替换为团队实际仓库地址。
- 若无访问权限，请先联系仓库管理员开通读取权限。

---

## 5. 基础设施准备

## 5.1 MySQL

创建数据库（用于初版部署）：

```sql
CREATE DATABASE IF NOT EXISTS uav_system DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 5.1.1 初始化数据结构与基础数据

在空库环境下，执行 `projects/backend-core/sql/database.sql` 完成表结构与基础数据初始化。

说明：

- 该脚本用于初版一次性初始化。
- 脚本中包含数据库重建语句，执行前请确认目标环境允许重建。

可通过以下 SQL 验证初始化结果：

```sql
SHOW TABLES;
```

## 5.2 Redis

确保 Redis 可连接，并准备好用户名/密码（若启用鉴权）。

## 5.3 MinIO

- 确保 MinIO 服务可用
- 创建目标 bucket（如 `uav-system`）
- 记录 endpoint/access-key/secret-key

---

## 6. 配置 backend-core

配置文件：`projects/backend-core/src/main/resources/application.yaml`

至少确认以下项与实际环境一致：

- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`
- `spring.data.redis.host`
- `spring.data.redis.port`
- `spring.data.redis.password`
- `minio.endpoint`
- `minio.access-key`
- `minio.secret-key`
- `minio.default-bucket`
- `server.port`
- `server.servlet.context-path`

---

## 7. 配置 yolo-service

配置文件：`projects/yolo-service/.env`

示例：

```ini
PROJECT_NAME="UAV Detection Service [DEV]"
APP_HOST=0.0.0.0
APP_PORT=8000
API_PREFIX="/api/v1"
DEBUG=true

MODEL_WEIGHTS_PATH="weights/yolo26n.pt"

UPLOAD_DIR="data/temp_uploads"
OUTPUT_DIR="data/outputs"
```

说明：

- 若使用 `.onnx`，请确保 `onnxruntime` 可用。
- 权重文件路径必须真实存在。

---

## 8. 配置 uav-frontend

配置文件：`projects/uav-frontend/.env`（如不存在可新建）

```ini
VITE_API_BASE_URL=http://localhost:8111/api/v1
```

如后端部署在远端，请替换为对应地址。

## 9. 本地开发启动

## 9.1 启动 yolo-service

```bash
cd projects/yolo-service
uv venv
uv sync
uv run python src/yoloservice/main.py
```

## 9.2 启动 backend-core

```bash
cd projects/backend-core
./mvnw -DskipTests compile
./mvnw spring-boot:run
```

Windows 可用：

```powershell
.\mvnw.cmd -DskipTests compile
.\mvnw.cmd spring-boot:run
```

## 9.3 启动 uav-frontend

```bash
cd projects/uav-frontend
npm install
npm run dev
```

## 9.4 启动后最小验收

建议按以下顺序检查（具体地址见 [9.5 统一访问地址](#95-统一访问地址)）：

1. `yolo-service` 健康接口
2. `backend-core` Swagger
3. 前端页面可访问
4. 前端登录后可进入节点管理与推理记录页面

## 9.5 统一访问地址

本地开发默认访问地址如下：

- 前端首页：`http://localhost:5173`
- `backend-core` Swagger：`http://localhost:8111/api/v1/swagger-ui/index.html`
- `yolo-service` Swagger：`http://localhost:8000/docs`
- `yolo-service` 健康检查：`http://localhost:8000/api/v1/health`

## 10. 构建与生产部署建议

## 10.1 前端

构建：

```bash
cd projects/uav-frontend
npm install
npm run build
```

产物目录：`projects/uav-frontend/dist`

可通过 Nginx 托管静态资源，并配置 SPA 回退规则（`try_files`）。

## 10.2 backend-core

打包：

```bash
cd projects/backend-core
./mvnw -DskipTests package
```

运行：

```bash
java -jar target/backend-core-0.0.1-SNAPSHOT.jar
```

建议：

- 通过环境变量或外置配置覆盖数据库和密钥信息
- 使用 systemd 管理进程

## 10.3 yolo-service

建议 systemd 或 Supervisor 托管：

```bash
cd projects/yolo-service
uv sync
uv run python src/yoloservice/main.py
```

生产建议：

- 固定 Python 环境与依赖版本
- 将模型权重与输出目录挂载到持久化卷
- 按硬件情况启用 GPU 推理

---

## 11. 联调验收清单

1. 前端可正常登录（JWT 流程通）
2. 节点管理页可拉取节点列表
3. 可上传/切换权重
4. 图片推理可返回结果 URL
5. 视频推理可返回结果 URL
6. 记录页可分页查询
7. 当存在处理中记录时，记录页自动轮询刷新
8. 推理状态显示符合：待处理/处理中/成功/失败

## 12. 常见问题与处理

## 12.1 前端构建失败（TypeScript 类型问题）

处理建议：

1. 先执行 `npm run build` 获取具体报错文件与行号
2. 优先修复页面类型定义（常见于表头类型与回调签名）
3. 修复后复测 `npm run build`

## 12.2 yolo-service 启动失败（模型加载）

检查项：

- `MODEL_WEIGHTS_PATH` 是否存在
- `.pt/.onnx` 依赖是否安装完整
- Python 虚拟环境是否正确

## 12.3 MinIO 上传成功但前端无法访问 URL

检查项：

- `minio.public-base-url` 是否可被浏览器访问
- bucket 策略与对象访问权限
- 反向代理是否透传正确

## 12.4 目录调整后命令仍报找不到路径

检查项：

- 当前工作目录是否在仓库根目录 `uav-system`
- 命令是否已使用 `projects/...` 前缀（如 `cd projects/backend-core`）
- IDE 任务脚本是否仍保留旧路径（如 `cd backend-core`）

---

## 13. 安全与运维建议

1. 禁止在仓库明文提交生产密码、JWT key、AccessKey。
2. 生产环境使用独立配置中心或环境变量注入敏感信息。
3. 数据库脚本与应用代码版本保持一致，部署前统一校验。
4. 对关键接口增加访问审计与限流策略。
5. 定期备份 MySQL、MinIO 对象与模型权重。

---

## 14. 推荐目录级运维命令

```bash
# 查看后端编译是否通过
cd projects/backend-core && ./mvnw -DskipTests compile

# 检查前端 i18n 完整性
cd projects/uav-frontend && npm run i18n:check

# 检查前端生产构建
cd projects/uav-frontend && npm run build

# 启动 yolo-service
cd projects/yolo-service && uv run python src/yoloservice/main.py
```
