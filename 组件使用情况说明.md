## 1. 文档目的

本文档面向 `uav-system` 初版项目，说明各子系统、模块、页面与接口的职责、调用关系与使用场景，帮助团队快速理解架构并开展开发与运维工作。

---

## 2. 项目整体架构

当前项目为前后端分离 + AI 推理微服务架构，主要由三部分组成（统一位于 `projects/` 目录下）：

1. `projects/uav-frontend`：Vue 3 前端管理台
2. `projects/backend-core`：Spring Boot 业务后端（统一鉴权、RBAC、业务编排）
3. `projects/yolo-service`：FastAPI 推理服务（实际调用 YOLO 模型）

### 2.1 请求链路

1. 前端发起业务请求（如图片推理、节点管理）到 `backend-core`
2. `backend-core` 完成鉴权、参数校验、业务持久化
3. 需要 AI 推理时，`backend-core` 转调 `yolo-service`
4. `yolo-service` 返回推理结果 URL/检测数据
5. `backend-core` 落库并回传给前端

### 2.2 关键状态定义

- 节点状态（Node）：用于表示节点可用性与运行状态，前端展示建议统一做状态归一化后再映射文案。
- 推理任务状态（Record）：`PENDING`、`RUNNING`、`SUCCESS`、`FAILED`，用于记录页筛选与轮询控制。

## 3. 子项目与技术栈

## 3.1 `uav-frontend`

- 框架：Vue 3 + TypeScript + Vite + Vuetify
- 路由：Vue Router
- 国际化：vue-i18n
- 构建脚本：
  - `dev`：本地开发
  - `build`：类型检查 + 生产构建
  - `preview`：构建产物预览
  - `i18n:check`：多语言完整性检查

### 3.1.1 主要页面模块（views）

- `LoginView.vue`：登录页
- `DashboardView.vue`：系统总览页
- `UserProfileView.vue`：个人中心
- `NodeManagementView.vue`：YOLO 节点管理（节点、权重、参数模板）
- `InferenceView.vue`：推理任务发起页（图片/视频）
- `RecordsView.vue`：推理记录页（分页、详情、删除、处理中轮询）
- `RbacAdminView.vue`：RBAC 管理页（管理员视图）
- `NotFoundView.vue`：404

### 3.1.2 通用模块（components）

- `PageHero.vue`：页面头部区域
- `AppSnackbar.vue`：全局提示条
- `ConfirmActionDialog.vue`：通用确认弹窗
- `PromptActionDialog.vue`：带输入的确认弹窗
- `InferenceNodeSelectorCard.vue`：推理节点选择卡片

### 3.1.3 路由与权限控制

路由文件：`projects/uav-frontend/src/router/index.ts`

- 公共路由：`/login`、`/not-found`
- 主路由：`/dashboard`、`/profile`、`/nodes`、`/inference`、`/records`、`/rbac`
- `inference` 路由开启 `keepAlive`，用于保留页面状态
- `rbac` 路由要求管理员角色（通过 `fetchMyProfile` 判断角色）

---

## 3.2 `backend-core`

- 框架：Spring Boot 4
- 数据层：MyBatis-Plus + MySQL
- 缓存：Redis
- 鉴权：Spring Security + JWT
- 对象存储：MinIO
- API 文档：springdoc-openapi
- 构建：Maven Wrapper (`mvnw.cmd`)

### 3.2.1 Controller 使用情况

- `AuthController.java`
  - 登录、刷新 token、登出
- `UserController.java`
  - 用户个人信息查询与修改
- `SysFileController.java`
  - 通用文件上传（含头像）
- `YoloNodeController.java`
  - 节点分页/详情/新增/更新/删除
  - 节点同步
  - 参数模板（增删改查、应用）
  - 权重（上传、切换、删除、列表）
- `YoloInferenceController.java`
  - 图片推理
  - 视频推理
  - 推理记录分页/详情/删除
- `AdminRbacController.java`
  - 角色、权限、用户角色关系管理

### 3.2.2 核心业务对象

- `YoloDetectionRecordEntity`
  - 推理记录实体
  - 关键字段：
    - `taskType`：任务类型（图像/视频）
    - `taskStatus`：任务状态（0 待处理、1 处理中、2 成功、3 失败）
    - `resultUrl`：推理结果地址
    - `detectionDetails`：检测结果 JSON

### 3.2.3 推理状态流转

在推理流程中，后端使用独立任务状态：

1. 创建记录：`PENDING`
2. 发起执行：`RUNNING`
3. 执行成功：`SUCCESS`
4. 执行失败：`FAILED`

该状态流转用于支持前端“处理中”状态展示与自动轮询刷新。

---

## 3.3 `yolo-service`

- 框架：FastAPI + Uvicorn
- 推理引擎：Ultralytics YOLO（支持 `.pt`、`.onnx`）
- 多媒体处理：OpenCV / moviepy
- 包管理：uv
- 主要接口：
  - `/api/v1/image/predict`
  - `/api/v1/video/predict`
  - `/api/v1/weights/*`
  - `/api/v1/params/*`
  - `/api/v1/health`

### 3.3.1 模型与参数管理能力

- 权重上传、删除、激活切换
- 推理参数读取与批量更新（如 `conf`、`iou`）

### 3.3.2 运行时可观测性

- 启动时打印 ultralytics 路径与关键运行时信息
- 启动脚本优先使用项目 `.venv` Python，减少解释器污染风险

## 4. 前后端功能映射关系

## 4.1 登录与用户信息

- 前端页面：`LoginView`、`UserProfileView`
- 后端接口：`/auth/*`、`/users/profile`

## 4.2 模型节点管理

- 前端页面：`NodeManagementView`
- 后端接口：`/yolo-nodes/*`
- 下游服务：`yolo-service` 权重与参数相关接口

## 4.3 推理任务

- 前端页面：`InferenceView`
- 后端接口：`/inference/nodes/{nodeId}/image|video`
- 下游服务：`/api/v1/image/predict`、`/api/v1/video/predict`

## 4.4 推理记录

- 前端页面：`RecordsView`
- 后端接口：`/inference/records/page`、`/inference/records/{id}`
- 状态展示：使用 `taskStatus`
- 轮询策略：仅当当前页存在处理中记录时启用

## 5. 数据与存储模块使用说明

## 5.1 MySQL

用于存储业务主数据：用户、角色权限、YOLO 节点、权重、参数模板、推理记录。

关键表：

- `users`
- `roles`
- `permissions`
- `user_roles`
- `role_permissions`
- `yolo_nodes`
- `yolo_weights`
- `yolo_node_params`
- `yolo_detection_records`

## 5.2 Redis

用于缓存、会话或 token 相关能力（由后端配置决定）。

## 5.3 MinIO

用于存储上传文件与推理结果媒体资源，后端/算法服务统一通过 URL 暴露给前端。

## 6. 配置项使用情况

## 6.1 前端

环境变量：

- `VITE_API_BASE_URL`：后端 API 基地址（默认 `http://localhost:8111/api/v1`）

## 6.2 后端 (`application.yaml`)

关键配置：

- `server.port`：默认 `8111`
- `server.servlet.context-path`：`/api/v1`
- `spring.datasource.*`：MySQL 连接
- `spring.data.redis.*`：Redis 连接
- `spring.security.jwt.*`：JWT 配置
- `minio.*`：MinIO 连接

## 6.3 yolo-service (`.env`)

关键配置：

- `APP_HOST` / `APP_PORT`
- `API_PREFIX`
- `MODEL_WEIGHTS_PATH`
- `UPLOAD_DIR` / `OUTPUT_DIR`
